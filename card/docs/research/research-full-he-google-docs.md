# מחקר מרוכז: משחק סלינדה — קושי, וריאנטים, טווח 0–10 מול 0–15 וקוביות

**מסמך לקריאה ולייבוא ל-Google Docs**  
**תאריך ריכוז:** אפריל 2026  

**הערת מוצר:** באפליקציה בפועל — מצב קל עם קלפים עד **0–12**, קוביות **1–6**.  
חלק מהמחקר המורחב נערך גם לפי תרחיש **קלפים 0–10** ו**קוביות 0–10** (להשוואה מחקרית); יש להבדיל בין ממצא מספרי להמלצת מוצר.

---

## סיכום מנהלים

1. **כניסה למתחילים:** מומלץ להתחיל ב־**0–12** עם **חיבור בלבד** (`+`), ואז להתקדם בצורה מדורגת לחיסור, לערבוב `+/−` בלי תוצאות שליליות, ורק אחר כך להרחיב טווח או להוסיף שליליים.
2. **מעבר ביניים:** **0–15** בלי כפל וחילוק הוא שלב טבעי — מגדיל את מרחב המספרים בלי להוסיף סוג חשיבה חדש (כפל/חילוק).
3. **כפל וחילוק:** מתאימים כ**הרחבה** אחרי שליטה בטווח צר, לא כנקודת פתיחה לקהל מתחיל רחב.
4. **שברים ו־0–25:** שכבת **מתקדמים**; עשיר למסגרות חינוך ולהעמקה, סיכון תסכול גבוה אם מוצג מוקדם מדי.
5. **מחקר 0–10 מול 0–15 (קוביות 0–10):** מעבר מ־0–10 ל־0–15 **מכפיל** כמעט את מספר תצורות התרגיל ב־`+/−` ללא שליליים; ממוצע יעדי קוביה גדל כשמאפשרים יעד עד 15; בסימולציה מפושטת **שברים** מגדילים מאוד את שיעור סיום המשחק לעומת חפיסה בלי שברים.

### מסקנות (מחקר מתחילים)

1. מעבר מ־0–12 ל־0–15 מגדיל משמעותית את עומס המתחיל.  
2. ערבוב `+/−` מעלה קושי גם בלי שליליים.  
3. שליליים מהווים קפיצת קושי חדה (כ־23% מהתרגילים במצבי D/H).  
4. לקהל מעורב: עדיף מסלול התקדמות מדורג.

### המלצה אופרטיבית

- ברירת מחדל התחלתית: **0–12**, **`+` בלבד**.  
- מסלול התקדמות מוצע:  
  1. 0–12, `+` בלבד  
  2. 0–12, `−` בלבד ללא שליליים  
  3. 0–12, `+/−` ללא שליליים  
  4. 0–15, `+` בלבד  
  5. 0–15, `+/−` ללא שליליים  
  6. שליליים — שלב מתקדם אופציונלי  

### תוכנית אימות מוצר (A/B) — הצעה

- משך: שבועיים.  
- קבוצה A: פתיחה 0–12 + בלבד. קבוצה B: פתיחה 0–15 + בלבד.  
- מדדים: דיוק, זמן תשובה, נטישה אחרי 10/20 תרגילים, קצב מעבר שלבים.  
- כלל: אם B לא פוגעת בנטישה/דיוק ביותר מ־5% ומביאה שיפור למידה מובהק — לשקול העלאת ברירת מחדל; אחרת להישאר ב־0–12.

### מפת מוצר מוצעת

- כניסה: **0–12** בלי ×÷.  
- התקדמות ראשונה: **0–15** בלי ×÷.  
- התקדמות שנייה: **0–12** או **0–15** עם ×÷.  
- מתקדם: **0–25** עם שברים (בלי או עם ×÷).  

### המלצה סופית (וריאנטים)

- ברירת מחדל: **0–12** בלי כפל וחילוק.

- וריאנטים עם כפל וחילוק כהרחבות.  
- **0–25** עם שברים כקו מתקדם, לא כגרסה בסיסית.

## חלק ג׳ — מחקר משלים: קלפים 0–10 ו־0–15, שלוש קוביות 0–10, רק +/−

### ג׳1 — מתודולוגיה (מרחב תרגילים דו־מספרי)

- **חיבור:** נשיאה כאשר `(a%10)+(b%10)≥10`.  
- **חיסור (ללא תוצאות שליליות בטבלה):** זוגות עם תוצאה כמקס־מין; השאלה כאשר `(max%10)<(min%10)`.  
- **+/− עם שליליים:** במחצית החיסור — מדד נשיאה/השאלה לפי `(a%10)<(b%10)` על כל הזוגות הסדורים.  
- **ציון עומס:** כיול לינארי מול שורות A–H של מחקר 0–12/0–15 (נוסחה מתועדת ב־JSON: שילוב אנטרופיה, אחוז שליליים, אחוז «קשים»).

## נספח — עדכון נתונים עתידי

מתוך תיקיית `card/`:

```bash
node scripts/research-exercise-space.js
node scripts/research-dice-enumeration.js

```

לאחר הרצה: לעדכן את הטבלאות בחלק ג׳ לפי הקבצים ב־`docs/research/research-0-10-15-exercise-metrics.json`, `docs/research/research-0-10-dice-enumeration.json`, `docs/research/research-0-10-15-simulation.json`.

### העברה ל-Google Docs

1. **מומלץ:** התקנת [Pandoc](https://pandoc.org/) במחשב והרצה מתוך תיקיית `card/`:  
   `pandoc docs/research/research-full-he-google-docs.md -o docs/research/research-full-he.docx`  
   לאחר מכן: Google Drive → העלאת הקובץ → פתיחה עם Google Docs.  
   *(בסביבת בנייה נבדקה לא הותקן Pandoc — אם אין פקודה כזו, השתמשו באפשרות 2.)*  
2. **ללא Pandoc:** פתיחת `docs/research/research-full-he-google-docs.md` בעורך טקסט או ב־VS Code, בחירה הכל → העתקה → הדבקה ב־Google Docs (ייתכן צורך ליישר טבלאות ידנית).
